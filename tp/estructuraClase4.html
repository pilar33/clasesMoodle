TALLER · Organización de la clase

Django inicial + MySQL + Migraciones + Git/GitHub

Duración: 2 h 30 min · Modalidad: práctica guiada

Objetivos

  • Verificar Python y crear/activar entorno virtual.
  • Instalar Django y crear proyecto + app.
  • Conectar a MySQL y ejecutar migraciones.
  • Inicializar repositorio, usar .gitignore y publicar en GitHub.

Agenda (con tiempos)

  • 0:00–0:05 · Apertura y objetivos.
  • 0:05–0:15 · Verificación de Python/pip y por qué usar venv.
  • 0:15–0:30 · Crear/activar venv e instalar Django.
  • 0:30–0:40 · Marco MVT (breve) y flujo URL → Vista → Modelo → Template.
  • 0:40–1:00 · startproject, startapp y estructura básica.
  • 1:00–1:20 · Conexión a MySQL (mysqlclient) y settings.DATABASES.
  • 1:20–1:35 · makemigrations / migrate y tablas creadas.
  • 1:35–2:15 · Git: .gitignore, primer commit, remoto y push a GitHub.
  • 2:15–2:30 · Cierre: dudas + checklist de entregables.

Actividades en clase

  • Crear y activar venv; instalar Django y comprobar runserver.
  • Crear proyecto django-admin startproject y app python manage.py startapp.
  • Instalar mysqlclient, configurar DATABASES y ejecutar migraciones.
  • Preparar .gitignore (excluir venv/, __pycache__/, .env, *.log).
  • Inicializar Git, primer commit y publicar en GitHub. Agregar a la docente como colaboradora.

Preguntas guía: ¿Qué tablas se crean al migrar? ¿Para qué sirven auth_user, auth_group?

Entregables

  • Repositorio en GitHub (con .gitignore) y acceso a la docente.
  • Captura de runserver o de la página de inicio.
  • Breve nota del grupo: qué ocurrió al ejecutar migrate (tablas creadas y propósito).

Materiales

  • Guía 1: Python/venv · Instalación Django · MVT · Proyecto/App (subida en la plataforma).
  • Guía 2: MySQL · mysqlclient · DATABASES · Migraciones · ORM básico (subida en la plataforma).
