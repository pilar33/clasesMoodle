TALLER · Clase 3

Git & GitHub – Plan de encuentro

Duración: 2 h – 2 h 30 min · Modalidad: práctica guiada

Objetivos

  • Entender el rol de Git (control de versiones) y GitHub (repositorio remoto).
  • Usar comandos básicos: init, clone, add, commit, push, pull.
  • Configurar un repo local y conectarlo con GitHub.
  • Practicar trabajo colaborativo y resolver un conflicto simple.

Requisitos Git instalado y cuenta activa en GitHub.

Contenidos

  • Flujo básico: Working Directory → Staging → Repository → Remote.
  • Comandos esenciales de Git (individual y colaborativo).
  • GitHub: creación de repos, remotos, permisos básicos.
  • Buenas prácticas de commits y mensajes.

Agenda

1) Intro breve (15–20’)

  • ¿Qué resuelve Git? ¿Qué aporta GitHub?
  • Diagrama del flujo local ↔ remoto.

2) Demo docente (20–25’)

  • git init, add, commit, remote, push.
  • Creación repo en GitHub y primer envío.

3) Práctica por grupos (60–75’)

  • Crear repo local, conectarlo con GitHub y hacer 2 commits por integrante.
  • Push/Pull entre compañeros.
  • Simular y resolver un conflicto simple.

4) Cierre (10’)

  • Repaso de comandos y buenas prácticas.
  • Entregables y próximos pasos.

Tip Mensajes de commit claros y en imperativo. Ej.: “Agrega validación de formulario de pago”.

Materiales

  • Presentación de la clase (slides del docente).
  • Guía práctica paso a paso (PDF/Doc). [enlace]
  • Cartilla resumida de comandos y flujo. [enlace]

Actividad en clase (checklist)

  • Configurar identidad: git config --global user.name / user.email.
  • Inicializar repo: git init y crear README.md.
  • Primer commit: git add + git commit -m.
  • Crear repo en GitHub y conectar: git remote add origin.
  • git push -u origin main (o la rama que usen).
  • Clonar desde otra máquina/compañero: git clone.
  • Pull/Push y conflicto simple → resolver y documentar.

Entregables

  • Repositorio en GitHub del grupo con mínimo 2 commits por integrante.
  • Un archivo con conflicto resuelto (evidencia en historial).
  • Captura del historial: git log --oneline o vista de commits en GitHub.
